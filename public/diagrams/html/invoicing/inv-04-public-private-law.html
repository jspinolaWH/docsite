PD-285 — Auto-classification criteria

  • Customer type (municipal vs commercial)
  • Product/service classification
  • Regional regulations
  • Contract terms

Classification is automatic — no manual tagging required for standard cases.

⚠️ PJH open question

PD-285 / PD-284: "Does the decision on the combined vs separate approach need to be made before go-live, and can the operating model be changed or selected after go-live?"

This is an open configuration decision that must be resolved during implementation.

Why this matters: different collection paths

Public-law invoices use statutory enforcement — the municipality can compel payment. Private-law invoices go through commercial debt collection. Using the wrong path for an invoice type creates legal compliance risk.
