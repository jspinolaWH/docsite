⚠️ PD-280 validation: If any subscriber is not allocated a share, the system notifies the invoicing user. If total shares ≠ 100% (even by 0.01%), the system blocks invoicing and alerts the user to correct the allocation.
✓ PD-279: Dynamic redistribution

When Neighbour E joins in May, the system re-generates events for the new period. The total service cost is always fully distributed — 100% collected across all periods. No manual recalculation needed.
